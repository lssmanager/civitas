# Security audit workflow

The `Security audit` workflow is intentionally limited to controls that the repository can execute and verify today. It covers the Node.js backend in `backend/` and the React/Vite frontend in `frontend/`.

## Controls

- **Dependency SCA:** each project installs dependencies with `npm ci` from its committed `package-lock.json` and runs `npm audit --audit-level=moderate --json`.
- **SAST:** CodeQL runs with the `javascript-typescript` language pack, which matches the JavaScript backend and TypeScript/React frontend in this repository.

## Supply-chain hardening

All GitHub Actions are pinned to immutable commit SHAs and annotated with the version tag that was resolved when the workflow was authored. Do not replace those SHAs with floating references such as `@main`, `@master`, or moving major tags like `@v4`.

## Evidence and fail-fast behavior

The workflow uploads only reports that were generated by the tools during the run:

- `npm-audit-backend.json`
- `npm-audit-frontend.json`
- CodeQL SARIF files from `codeql-results/`

Report creation is mandatory. A missing or empty npm audit report, invalid npm audit JSON, or missing CodeQL SARIF file fails the workflow instead of continuing silently. The npm audit job uploads the JSON report before failing on vulnerabilities at or above the configured `moderate` threshold so reviewers can inspect the evidence.

## Non-goals

This workflow does not claim ISO 27001, NIST CSF, SSDF, SLSA, or MITRE ATT&CK compliance by itself. Those frameworks can be mapped only from real findings and additional process evidence; this pipeline does not generate simulated compliance reports.
